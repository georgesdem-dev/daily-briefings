Miasma Worm Hits 73 Microsoft GitHub Repositories via AI Coding Agents
A self-replicating worm dubbed "Miasma" — a variant of the Mini Shai-Hulud worm publicly released by TeamPCP in May — compromised 73 repositories across Azure, Azure-Samples, Microsoft, and MicrosoftDocs. Entry vector: a malicious commit pushed to Azure/durabletask using previously stolen contributor credentials. The commit injected configuration files that trigger a credential-harvesting payload when any AI coding tool (Claude Code, Cursor, Gemini CLI, VS Code) opens the repo. The worm then harvests AWS, Azure, GCP, Kubernetes, npm, and GitHub tokens and self-propagates to any repo the victim can write to. GitHub disabled all 73 repos within 105 seconds of detection. Developers should audit recently cloned Microsoft repos and rotate any cloud/CI credentials.
